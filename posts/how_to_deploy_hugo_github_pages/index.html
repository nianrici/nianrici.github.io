Le Blog

Cómo crear un blog usando Hugo y Github Pages

En esta entrada voy a explicar como montar un blog como este. Si has llegado hasta aquí, es porque no te hace falta mucha más información, ni necesitas que te vendan el método. Casi seguro que sólo quieres que te digan el cómo para comparar si te renta hacerlo así o si quieres/necesitas algo diferente.

  • Crear 2 repositorios en Github:

    • Un repositorio llamado {$github_username}.github.io

Instalar Hugo en el PC

Esto dependerá del sistema operativo o de la distribución que uses. En mi caso, al usar una distro GNU Linux basada en Archlinux, sólo tendré que hacer:

sudo pacman -Syu && sudo pacman -S hugo

Otra opción más genérica sería clonar el repositorio de github y seguir las instrucciones de los creadores:

git clone https://github.com/gohugoio/hugo.git

Una vez instalado, procederemos con los siguientes pasos.

Crear 2 repositorios en Github:

Ahora toca coger el navegador, ir hasta nuestra página de Github y crear 2 repositorios: uno para almacenar el código generado por Hugo y el otro para deployear el blog.

Un repositorio llamado Blog

Crearemos el repositorio donde almacenaremos el código del blog (los archivos en Markdown, nuestros pequeños upgrades a los temas originales, etc…) y en un alarde de imaginación, lo llamaremos blog, para que no nos liemos.

Un repositorio llamado {$github_username}.github.io

Una vez creado el repositorio blog crearemos el otro, que será donde al final subamos los archivos estáticos generados por Hugo. El nombre será nuestro usuario de github y el dominio .github.io (piensa que si no dispones de un dominio propio, esta será la URL definitiva, por lo que no compliques el tema demasiado)

Clonar el repositorio Blog en el PC

Sin más misterios, clonaremos nuestro repositorio blog en nuestro PC:

git clone https://github.com/$github_username/blog.git

Nota: ya se que no hace falta que lo diga, pero cuando pongo $github_username quiero decir tu nombre de tu usuario de Github tuyo propio único e intransferible…

Crear el blog dentro de la carpeta del repositorio

Ya clonado el repositorio del blog en local, pasamos a crear el blog con Hugo.

hugo new site nombredelblog

Este comando creará la estructura de carpetas y los archivos mínimos para montar el blog. Si quieres saber más sobre la estructura que usa Hugo, en su página web encontrarás documentación por un tubo, bien explicada y organizada.

Seleccionar un tema e instalarlo (opcional, pero recomendable)

Con el tema por defecto, se puede ir tirando… pero si en realidad quieres hacer algo diferente y menos genérico, toca buscar, comparar y descargar uno de los múltiples temas que encontrarás por la red.

Ir a la web de los temas

Aquí hay una lista de temas gratuitos, con sus instrucciones y todo.

Instalar el tema

Los pasos a seguir pueden ir desde hacer un wget/curl de una carpeta remota a clonar un repositorio de github en el que se encuentre alojado el tema.

Ya instalado, puedes meter las manos bajo el capó y modificar los .css, la estructura de las plantillas, etc… siempre y cuando sepas lo que haces y que no le temas a tener que volver a instalar el tema de nuevo (lo que implicaría como mucho, borrar la carpeta del tema y volver a clonar el repositorio desde donde lo descargaste en un principio).

Configuración del Blog (config.toml)

El archivo config.toml contiene información de la configuración relevante para la página, tal como:

baseURL = "https://foo.com/"
languageCode = "es-es"
title = "My Hugo Site"
theme = "TheTheme"

Esta información es la que viene por defecto y deberá ser reemplazada por la nuestra.

Creación de la primera entrada

La lógica nos dice que lo suyo sería crear el archivo directamente en la carpeta de content y puerta (que podría hacerse), pero para no dejarnos nada en la cabecera del archivo que contiene la entrada, la crearemos así:

hugo new content/primera_entrada.md

Esto creará el archivo en la carpeta correspondiente, creando la cabecera con información de la autoría, taxonomía, etiquetas o cualquiera de las múltiples opciones que se pueden añadir, además de las estándard y que podrá ser editado usando el editor que se desee (vim, emacs, notepad++, etc…) y en el que se usará el lenguaje por marcas Markdown, pero la versión de Github (si, hay varias versiones, pero esta está ampliamente documentada).

Preview

Una vez rellenada la primera entrada, ejecutaremos el siguiente comando, el cual montará un servidor que sólo mostrará las entradas que estén marcadas en la cabecera como draft = true.

hugo server -D

Accederemos a localhost:1313 para ver como queda, y así poder proceder a cambiar la sintaxis, los css del tema (o cambiar el tema si procediese), etc…

Uso de los repos

Vale, ya tenemos la entrada preparada, nos gusta como queda todo y queremos presentársela al mundo. Estos son los pasos a seguir:

Repo de la url del blog

Empezaremos con el repositorio con la url y vamos a:

  1. Clonar el repositorio
git clone https://github.com/$github_username/$github_username.github.io.git
  1. Crear una rama “main”
cd  $github_username.github.io
git checkout -b main
  1. Generar un README.md (puede estar vacío)
touch README.md
  1. Crear el primer commit
git add .
git commit -m "añado readme"
  1. Subirlo (git push)
git push origin main

Hecho esto, seguiremos con el repositorio donde almacenaremos el código.

Carpeta del blog

Iremos a la carpeta generada por hugo con el blog y donde están las carpetas de Hugo y el archivo config.toml y seguiremos los siguientes pasos (venga, que ya queda poco):

  1. Creamos un submódulo del repo con la url del blog, que enlazará la carpeta public, que es donde se genera la página una vez Hugo la procesa.
git submodule add -b main  https://github.com/$github_username/$github_username.github.io.git public
  1. Generar los archivos estáticos (ojito, hay que quitar la opción de draft = true de la cabezera del/los post/s antes de ejecutar el siguiente paso!)
hugo -D
  1. Si todo ha ido como debería, verificar los ficheros creados en la carpeta public (que será la que subamos al repo con la url del blog y que contendrá la estructura final del sitio)
cd public
ls

Ahí deberían aparecer cosas tales como un index.html, varias carpetas con nombres lógicos dentro de una estructura de carpetas de una web, etc… 4. Subimos el contenido de la carpeta Public

git add .
git commit -m "commit inicial"
git push origin main

Verificamos el trabajo realizado

Para finiquitar, miraremos que el repositorio de contenga lo mismo que la carpeta public y si nos parece correcto, ya podríamos visitar nuestro blog en la url/public
